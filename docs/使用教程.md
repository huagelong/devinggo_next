## 项目简介

DevingGo 是一个基于 **GoFrame v2 + Vue3 + Arco Design** 开发的全栈前后端分离的后台管理系统。

### 核心特性

- ✅ **现代化技术栈**：GoFrame v2、PostgreSQL、Redis、Vue3、Vite5
- ✅ **模块化设计**：支持模块管理，易于扩展
- ✅ **多语言支持**：内置国际化（i18n）支持
- ✅ **多主题切换**：支持动态主题配置
- ✅ **队列系统**：基于 Redis 的异步任务队列
- ✅ **WebSocket**：支持实时通信和分布式部署
- ✅ **数据库迁移**：Migrate 工具管理数据库版本
- ✅ **接口文档**：自动生成 Swagger 文档
- ✅ **Docker部署**：支持容器化部署

### 功能模块

- 👤 用户管理：用户增删改查、权限分配
- 🏢 部门管理：树形结构组织管理
- 💼 岗位管理：职务配置
- 📋 菜单管理：动态菜单、按钮权限
- 🛡️ 角色管理：角色权限、数据权限
- 📚 字典管理：系统字典维护
- ⚙️ 参数管理：系统参数配置
- 📢 通知公告：公告发布管理
- 📝 操作日志：系统操作记录
- 🔐 登录日志：登录行为追踪
- 🔧 代码生成：CRUD代码生成器
- 📊 系统监控：Redis、在线用户监控
- 🔌 模块管理：系统模块管理
- ⏰ 定时任务：Cron任务调度
- 📎 附件管理：文件上传管理

## 安装指南

### Windows系统

#### 1. 安装构建工具

打开 **PowerShell（管理员权限）** 执行：

```powershell
# 安装 Chocolatey 包管理器
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 安装 Make 和 Sed
choco install make -y
choco install sed -y
```

#### 2. 安装开发环境

```powershell
# 安装 Go
choco install golang -y

# 安装 Node.js
choco install nodejs -y

# 验证安装
go version
node -v
npm -v
```

#### 3. 安装数据库

**PostgreSQL:**

```powershell
# 使用 Chocolatey 安装
choco install postgresql -y

# 或手动下载安装包
# https://www.postgresql.org/download/windows/
```

**Redis:**

```powershell
# 方式1: 使用 Memurai（Windows 原生）
choco install memurai-developer -y

# 方式2: 使用 WSL2 + Docker
# 或直接下载 Redis for Windows
# https://github.com/tporadowski/redis/releases
```

### Linux/macOS系统

#### Ubuntu/Debian

```bash
# 安装基础工具
sudo apt update
sudo apt install -y make git curl

# 安装 Go
wget https://go.dev/dl/go1.23.4.linux-amd64.tar.gz

echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

sudo apt install -y postgresql postgresql-contrib
```

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

## 快速开始

### 1. 克隆项目

```bash
# 克隆主仓库
git clone https://github.com/huagelong/devinggo.git
cd devinggo

# 拉取子模块（前端代码）
git submodule update --init --recursive
git submodule foreach git pull origin master
```

### 2. 配置数据库

#### 创建数据库

```sql
-- 连接 PostgreSQL
psql -U postgres

-- 创建数据库
CREATE DATABASE devinggo;

-- 退出
\q
```

### 3. 配置文件

#### 开发环境配置

```bash
# 复制开发配置文件
cp hack/config.example.yaml hack/config.yaml

# 编辑配置文件
# Windows: notepad hack/config.yaml
# Linux/macOS: vim hack/config.yaml
```

修改 `hack/config.yaml` 中的数据库和 Redis 配置：

```yaml
database:
  default:
    link: "pgsql:host=127.0.0.1 port=5432 dbname=devinggo user=postgres password=你的密码 sslmode=disable TimeZone=Asia/Shanghai"

redis:
  default:
    address: 127.0.0.1:6379
    pass: "你的Redis密码"  # 如果没有密码可以留空 ""
    db: 0
```

#### 生产环境配置

```bash
# 复制生产配置文件
cp manifest/config/config.example.yaml manifest/config/config.yaml

# 修改配置（同上）
```

### 4. 安装依赖

```bash
# 安装 Go 依赖
go mod tidy

# 安装前端依赖（进入前端子模块）
cd web/admin
npm install
cd ../..
```

### 5. 初始化数据库

```bash
# 运行数据库迁移（创建表和初始数据）
go run main.go migrate:up
```

### 6. 启动项目

#### 后端启动

```bash
# 开发模式（热重载）
go run main.go

# 或使用 Make 命令
make run
```

后端启动成功后，访问：
- 🌐 **API服务**: http://localhost:8070
- 📘 **接口文档**: http://localhost:8070/swagger

#### 前端启动

```bash
# 进入前端目录
cd web/admin

# 启动开发服务器
npm run dev
```

前端启动成功后，访问：
- 🎨 **管理后台**: http://localhost:5173/admin

### 7. 登录系统

**默认账号:**
- 用户名: `superAdmin`
- 密码: `admin123`

## 配置说明

### 服务器配置 (server)

```yaml
server:
  name: "devinggo_1"        # 服务名称，WebSocket分布式需要区分
  address: ":8070"          # 监听地址和端口
  serverRoot: "resource/public"  # 静态文件根目录
  openapiPath: "/api.json"  # OpenAPI 文档路径
  swaggerPath: "/swagger"   # Swagger UI 路径
  
  # 请求限制
  clientMaxBodySize: "20MB" # 最大请求体大小
  maxHeaderBytes: "10240"   # 最大请求头大小
  
  # 日志配置
  logPath: "logs/"          # 日志目录
  accessLogEnabled: true    # 访问日志
  errorLogEnabled: true     # 错误日志
```

### 数据库配置 (database)

```yaml
database:
  logger:
    path: "logs/"
    file: "sql-{Ymd}.log"   # SQL日志文件
    level: "all"            # 日志级别
    stdout: true            # 终端输出
  
  default:
    link: "pgsql:host=127.0.0.1 port=5432 dbname=devinggo user=postgres password=123456 sslmode=disable TimeZone=Asia/Shanghai"
    debug: true             # 开启SQL调试
```

### Redis配置 (redis)

```yaml
redis:
  # 默认Redis（业务缓存）
  default:
    address: 127.0.0.1:6379
    pass: "123456"
    db: 0
    maxConnLifetime: 30m
    idleTimeout: 10m
  
  # 缓存专用Redis
  cache:
    address: 127.0.0.1:6379
    pass: "123456"
    db: 1
  
  # WebSocket专用Redis（分布式通信）
  websocket:
    address: 127.0.0.1:6379
    pass: "123456"
    db: 7
```

### 队列配置 (worker)

```yaml
worker:
  location: "Asia/Shanghai"     # 时区
  shutdownTimeout: 30s          # 优雅关闭超时
  concurrency: 20               # 并发数
  queues:                       # 队列优先级
    - critical: 6               # 高优先级
    - default: 3                # 中优先级
    - low: 1                    # 低优先级
  redis:
    address: 127.0.0.1:6379
    pass: "123456"
    db: 2
```

### 上传配置 (upload)

```yaml
upload:
  dir: "uploads"  # 上传目录（相对于 resource/public）
```

## 基础使用

### 用户管理

#### 1. 添加用户

- 进入【系统管理】→【用户管理】
- 点击【新增】按钮
- 填写用户信息：
  - 用户名（必填）
  - 姓名
  - 密码
  - 所属部门
  - 岗位
  - 角色
- 点击【确定】保存

#### 2. 分配角色

- 在用户列表找到目标用户
- 点击【编辑】
- 在角色下拉框中选择角色
- 点击【确定】保存

#### 3. 重置密码

- 点击用户操作栏的【重置密码】
- 系统将密码重置为默认密码（在系统参数中配置）

### 菜单管理

#### 1. 添加菜单

- 进入【系统管理】→【菜单管理】
- 点击【新增】按钮
- 填写菜单信息：
  - **菜单类型**: 目录/菜单/按钮
  - **菜单名称**: 显示名称
  - **路由地址**: 前端路由路径
  - **组件路径**: Vue组件路径（菜单类型需要）
  - **权限标识**: 后端权限码（如：`system:user:add`）
  - **排序**: 显示顺序

#### 2. 菜单类型说明

- **目录（Catalog）**: 一级菜单，可包含子菜单
- **菜单（Menu）**: 具体页面，对应一个Vue组件
- **按钮（Button）**: 页面内按钮权限

### 角色管理

#### 1. 创建角色

- 进入【系统管理】→【角色管理】
- 点击【新增】
- 填写角色信息：
  - 角色名称
  - 角色标识（英文，如：`admin`）
  - 角色备注

#### 2. 分配权限

- 点击角色的【菜单权限】按钮
- 勾选允许该角色访问的菜单和按钮
- 点击【确定】保存

#### 3. 数据权限

- 点击角色的【数据权限】按钮
- 选择数据权限范围：
  - 全部数据权限
  - 自定义数据权限
  - 本部门数据权限
  - 本部门及以下数据权限
  - 仅本人数据权限

### 部门管理

#### 1. 部门结构

- 进入【系统管理】→【部门管理】
- 树形结构展示组织架构
- 支持拖拽排序

#### 2. 添加部门

- 点击【新增】或选中父部门后点击【新增子部门】
- 填写部门信息：
  - 部门名称
  - 负责人
  - 联系电话
  - 邮箱
  - 排序

### 字典管理

#### 1. 字典类型

- 进入【系统管理】→【字典管理】
- 点击【新增字典】创建字典类型
- 填写：
  - 字典名称（如：用户状态）
  - 字典类型（如：`user_status`）

#### 2. 字典数据

- 选中字典类型后点击【字典数据】
- 添加具体数据项：
  - 数据标签（显示值）
  - 数据键值（实际值）
  - 排序

### 定时任务

#### 1. 创建定时任务

- 进入【系统监控】→【定时任务】
- 点击【新增】
- 填写任务信息：
  - **任务名称**: 描述性名称
  - **任务标识**: 唯一标识（如：`sync_user_data`）
  - **Cron表达式**: 如 `0 0 * * *`（每天零点）
  - **任务类型**: 系统任务/自定义任务
  - **调用目标**: 处理函数名

#### 2. Cron表达式示例

```
# 每分钟执行
*/1 * * * *

# 每小时执行
0 * * * *

# 每天凌晨2点执行
0 2 * * *

# 每周一上午10点执行
0 10 * * 1

# 每月1号凌晨执行
0 0 1 * *
```

### 队列任务

#### 1. 创建队列任务

```go
package logic

import (
    "context"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/hibiken/asynq"
)

// 推送队列任务
func PushTask(ctx context.Context, userID int64) error {
    client := asynq.NewClient(asynq.RedisClientOpt{
        Addr: "127.0.0.1:6379",
        Password: "123456",
        DB: 2,
    })
    defer client.Close()
    
    // 创建任务
    task := asynq.NewTask(
        "email:send",
        []byte(`{"user_id": `+gconv.String(userID)+`}`),
    )
    
    // 推送到队列
    info, err := client.Enqueue(
        task,
        asynq.Queue("default"),  // 队列名称
        asynq.MaxRetry(3),       // 重试次数
    )
    
    if err != nil {
        return err
    }
    
    g.Log().Infof(ctx, "任务已推送: %s", info.ID)
    return nil
}
```

#### 2. 延迟队列

```go
// 10分钟后执行
task := asynq.NewTask("order:cancel", payload)
client.Enqueue(task, asynq.ProcessIn(10*time.Minute))

// 定时执行
client.Enqueue(task, asynq.ProcessAt(time.Date(2026, 1, 1, 0, 0, 0, 0, time.UTC)))
```

### WebSocket

#### 1. 前端连接

```javascript
// 建立连接
const ws = new WebSocket('ws://localhost:8070/ws')

ws.onopen = () => {
  console.log('WebSocket连接成功')
  
  // 发送消息
  ws.send(JSON.stringify({
    type: 'ping',

  }))

ws.onmessage = (event) => {
  const data = JSON.parse(event.data)

  console.log('WebSocket连接关闭')
}

ws.onerror = (error) => {
  console.error('WebSocket错误:', error)

```

#### 2. 后端推送

```go
package logic

import (
    "context"
    "github.com/gogf/gf/v2/frame/g"
)

// 推送消息给指定用户
func PushToUser(ctx context.Context, userID int64, message string) error {
    // 使用 Redis 发布订阅
    _, err := g.Redis().Publish(ctx, "ws:user:"+gconv.String(userID), message)
    return err
}

// 广播消息给所有用户
func BroadcastMessage(ctx context.Context, message string) error {
    _, err := g.Redis().Publish(ctx, "ws:broadcast", message)
    return err
}
```

### 创建新模块

#### 1. 模块结构

```
modules/
└── your_module/         # 你的模块名
    ├── module.go       # 模块定义
    ├── module.json     # 模块信息
    ├── api/            # API接口定义
    ├── controller/     # 控制器
    ├── logic/          # 业务逻辑
    ├── model/          # 数据模型
    ├── router/         # 路由
    ├── service/        # 服务接口
    └── worker/         # 队列任务
```

#### 2. module.json 配置

```json
{
  "name": "your_module",
  "title": "你的模块",
  "description": "模块描述",
  "version": "1.0.0",
  "author": "Your Name",
  "install": true,
  "enable": true
}
```

#### 3. module.go 实现

```go
package your_module

import (
    "context"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/net/ghttp"
)

type Module struct{}

func init() {
    // 注册模块
    g.Server().BindHandler("/your_module/install", Install)
    g.Server().BindHandler("/your_module/uninstall", Uninstall)
}

// Install 安装模块
func Install(ctx context.Context, req *ghttp.Request) {
    // 执行安装逻辑
    // 1. 创建数据表
    // 2. 初始化数据
    // 3. 注册路由
    
    req.Response.WriteJson(g.Map{
        "code": 0,
        "message": "模块安装成功",
    })
}

// Uninstall 卸载模块
func Uninstall(ctx context.Context, req *ghttp.Request) {
    // 执行卸载逻辑
    // 1. 删除数据表
    // 2. 清理数据
    
    req.Response.WriteJson(g.Map{
        "code": 0,
        "message": "模块卸载成功",
    })
}
```

#### 4. 注册路由

```go
package router

import (
    "your_module/controller"
    "github.com/gogf/gf/v2/net/ghttp"
)

func Register(group *ghttp.RouterGroup) {
    group.Group("/api/v1/your_module", func(group *ghttp.RouterGroup) {
        // 中间件
        group.Middleware(/* 认证中间件 */)
        
        // 路由绑定

            controller.User,      // 用户控制器

        )
    })
```

### 数据库操作

#### 1. 基础查询

```go
package logic

import (
    "context"
    "devinggo/internal/dao"
    "github.com/gogf/gf/v2/frame/g"
)

// 查询单条
func GetUserById(ctx context.Context, id int64) (*entity.SystemUser, error) {
    var user *entity.SystemUser
    err := dao.SystemUser.Ctx(ctx).Where("id", id).Scan(&user)
    return user, err
}

// 查询列表
func GetUserList(ctx context.Context, page, pageSize int) ([]*entity.SystemUser, error) {
    var users []*entity.SystemUser
    err := dao.SystemUser.Ctx(ctx).
        Page(page, pageSize).
        Order("id desc").
        Scan(&users)
    return users, err
}

// 条件查询
func SearchUsers(ctx context.Context, keyword string) ([]*entity.SystemUser, error) {
    var users []*entity.SystemUser
    err := dao.SystemUser.Ctx(ctx).
        WhereLike("username", "%"+keyword+"%").
        OrWhereLike("nickname", "%"+keyword+"%").
        Scan(&users)
    return users, err
}
```

```go
// 插入单条
func CreateUser(ctx context.Context, data *model.UserCreateInput) (int64, error) {
```

```go
    _, err := dao.SystemUser.Ctx(ctx).
```

```go
        Delete()
    return err
}

// 软删除（需要表有 deleted_at 字段）

    _, err := dao.SystemUser.Ctx(ctx).

        Data(g.Map{"deleted_at": gtime.Now()}).
        Update()
    return err
}
```

#### 5. 事务处理

```go
func TransferMoney(ctx context.Context, fromUser, toUser int64, amount float64) error {
    return g.DB().Transaction(ctx, func(ctx context.Context, tx gdb.TX) error {
        // 扣款
        _, err := tx.Model("account").
            Where("user_id", fromUser).
            Decrement("balance", amount)
        if err != nil {
            return err
        }
        
        // 加款
        _, err = tx.Model("account").
            Where("user_id", toUser).
            Increment("balance", amount)
        if err != nil {
            return err
        }
        
        return nil
    })
}
```

### 缓存使用

```go
package logic

import (
    "context"
    "time"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/os/gcache"
)

// 设置缓存
func SetCache(ctx context.Context, key string, value interface{}, expire time.Duration) error {
    return g.Redis().Set(ctx, key, value, gcache.SetOption{
        Duration: expire,
    })
}

// 获取缓存
func GetCache(ctx context.Context, key string) (interface{}, error) {
    return g.Redis().Get(ctx, key)
}

// 删除缓存
func DeleteCache(ctx context.Context, keys ...string) error {
    _, err := g.Redis().Del(ctx, keys...)
    return err
}

// 缓存+数据库模式
func GetUserWithCache(ctx context.Context, id int64) (*entity.SystemUser, error) {
    cacheKey := "user:" + gconv.String(id)
    
    // 先查缓存
    cacheData, err := g.Redis().Get(ctx, cacheKey)
    if err == nil && !cacheData.IsEmpty() {
        var user *entity.SystemUser
        _ = cacheData.Struct(&user)
        return user, nil
    }
    
    // 缓存未命中，查数据库
    user, err := GetUserById(ctx, id)
    if err != nil {
        return nil, err
    }
    
    // 写入缓存
    _ = g.Redis().Set(ctx, cacheKey, user, gcache.SetOption{
        Duration: 5 * time.Minute,
    })
    
    return user, nil
}
```

### Docker部署

#### 1. 构建镜像

```bash
# 构建后端镜像
docker build -t devinggo:latest .

# 或使用 Make
make docker
```

#### 2. Docker Compose 部署

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  # PostgreSQL 数据库
  postgres:
    image: postgres:15
    container_name: devinggo_postgres
    environment:
      POSTGRES_DB: devinggo
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: your_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    networks:
      - devinggo_network

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: devinggo_redis
    command: redis-server --requirepass your_redis_password
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    networks:
      - devinggo_network

  # 后端服务
  app:
    image: devinggo:latest
    container_name: devinggo_app
    depends_on:
      - postgres
      - redis
    environment:
      - GF_DATABASE_DEFAULT_LINK=pgsql:host=postgres port=5432 dbname=devinggo user=postgres password=your_password sslmode=disable TimeZone=Asia/Shanghai
      - GF_REDIS_DEFAULT_ADDRESS=redis:6379
      - GF_REDIS_DEFAULT_PASS=your_redis_password
    volumes:
      - ./logs:/app/logs
      - ./resource/public/uploads:/app/resource/public/uploads
    ports:
      - "8070:8070"
    networks:
      - devinggo_network
    restart: unless-stopped

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: devinggo_nginx
    depends_on:
      - app
    volumes:
      - ./docs/docker/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./web/admin/dist:/usr/share/nginx/html:ro
    ports:
      - "80:80"
      - "443:443"
    networks:
      - devinggo_network
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:

networks:
  devinggo_network:
    driver: bridge
```

启动服务：

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f app

# 停止服务
docker-compose down
```

### Nginx 配置

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    
    # 前端静态文件
    location /admin {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }
    
    # 后端API代理
    location /api {
        proxy_pass http://app:8070;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # WebSocket 代理
    location /ws {
        proxy_pass http://app:8070;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
    
    # 上传文件访问
    location /uploads {
        alias /app/resource/public/uploads;
        expires 30d;
    }
}
```

### Systemd 服务

创建服务文件 `/etc/systemd/system/devinggo.service`：

```ini
[Unit]
Description=DevingGo Service
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=www
Group=www
WorkingDirectory=/var/www/devinggo
ExecStart=/var/www/devinggo/bin/devinggo
Restart=on-failure
RestartSec=5s

# 环境变量
Environment="GF_ENV=prod"

# 日志
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

## 常见问题

### Q1: 数据库连接失败

**错误信息**: `connect: connection refused`

**解决方案**:
1. 检查 PostgreSQL 是否启动：
   ```bash
   # Linux
   sudo systemctl status postgresql
   
   # Windows
   services.msc 查看 PostgreSQL 服务
   ```

2. 检查配置文件中的数据库地址和端口
3. 确认数据库已创建：
   ```sql
   CREATE DATABASE devinggo;
   ```

4. 测试连接：
   ```bash
   psql -h 127.0.0.1 -p 5432 -U postgres -d devinggo
   ```

### Q2: Redis 连接失败

**错误信息**: `dial tcp 127.0.0.1:6379: connect: connection refused`

**解决方案**:
1. 启动 Redis：
   ```bash
   # Linux
   sudo systemctl start redis
   
   # Windows
   redis-server.exe
   ```

2. 检查 Redis 密码配置是否正确
3. 测试连接：
   ```bash
   redis-cli -h 127.0.0.1 -p 6379 -a your_password ping
   ```

### Q3: 迁移失败

**错误**: `migration failed: relation "xxx" already exists`

**解决方案**:
```bash
# 回滚所有迁移
go run main.go migrate:down

# 重新执行迁移
go run main.go migrate:up
```

### Q4: 前端访问404

**原因**: 前端资源未编译或路径不正确

**解决方案**:
```bash
cd web/admin
npm run build

# 确保 nginx 配置正确指向 dist 目录
```

### Q5: 跨域问题

**错误**: `Access-Control-Allow-Origin`

**解决方案**:在后端配置跨域中间件（通常已内置），或在 Nginx 添加：

```nginx
add_header Access-Control-Allow-Origin *;
add_header Access-Control-Allow-Methods 'GET, POST, PUT, DELETE, OPTIONS';
add_header Access-Control-Allow-Headers 'DNT,X-Mx-ReqToken,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Authorization';
```

### Q6: 上传文件失败

**错误**: `file size exceeds limit`

**解决方案**:
1. 检查服务器配置：
   ```yaml
   server:
     clientMaxBodySize: "20MB"  # 增加限制
   ```

2. 检查 Nginx 配置：
   ```nginx
   client_max_body_size 20M;
   ```

### Q7: 定时任务不执行

**排查步骤**:
1. 检查任务是否启用
2. 检查 Cron 表达式是否正确
3. 查看系统日志：
   ```bash
   tail -f logs/worker/*.log
   ```

### Q8: 内存占用过高

**优化方案**:
1. 检查是否有 goroutine 泄漏
2. 优化数据库查询，避免一次加载大量数据
3. 合理设置连接池大小
4. 使用分页查询

### Q9: 编译错误

**错误**: `cannot find module`

**解决方案**:
```bash
# 清理模块缓存
go clean -modcache

# 重新下载依赖
go mod tidy
go mod download
```

### Q10: 前端白屏

**排查步骤**:
1. 打开浏览器控制台查看错误
2. 检查 API 地址配置是否正确
3. 检查后端是否正常运行
4. 清除浏览器缓存
