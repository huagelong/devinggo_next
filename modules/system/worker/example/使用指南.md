# Worker优化使用指南

## 概述

新版本提供了极简的Worker、Cron和Task创建方式，大幅减少样板代码，提升开发效率。

## 代码量

| 类型 | 代码量 |
|-----|-------|
| Worker | ~1行注册 |
| Cron | ~1行注册 |
| Task | Builder链式调用 |

---

## 1. Worker注册

### 方式1: 函数式注册（推荐）

```go
package server

import (
    "context"
    "devinggo/modules/system/pkg/worker"
    glob2 "devinggo/modules/system/pkg/worker/glob"
    "github.com/hibiken/asynq"
)

type MyWorkerData struct {
    Message string `json:"message"`
}

func init() {
    // 直接注册执行函数，只需1行
    worker.RegisterWorkerFunc("my:worker:type", executeMyWorker)
}

func executeMyWorker(ctx context.Context, t *asynq.Task) error {
    data, err := glob2.GetParamters[MyWorkerData](ctx, t)
    if err != nil {
        return err
    }
    glob2.WithWorkLog().Infof(ctx, "处理数据: %s", data.Message)
    return nil
}
```

### 方式2: Builder模式（更灵活）

```go
func init() {
    worker.NewWorkerBuilder("my:worker:type").
        WithExecute(executeMyWorker).
        Register()
}
```

## 2. Cron注册

### 方式1: 函数式注册（推荐）

```go
package cron

import (
    "context"
    "devinggo/modules/system/pkg/worker"
    glob2 "devinggo/modules/system/pkg/worker/glob"
    "github.com/gogf/gf/v2/encoding/gjson"
    "github.com/gogf/gf/v2/frame/g"
)

type MyCronData struct {
    Name string `json:"name"`
}

func init() {
    // 直接注册，只需1行
    worker.RegisterCronFunc("my:cron:type", "我的定时任务", handleMyCronParams)
}

func handleMyCronParams(ctx context.Context, payload *glob2.Payload, params *gjson.Json) {
    if g.IsEmpty(params) {
        return
    }
    data := new(MyCronData)
    if err := params.Scan(data); err != nil {
        glob2.WithWorkLog().Errorf(ctx, "解析参数失败: %v", err)
        return
    }
    payload.Data = data
}
```

### 方式2: Builder模式（更灵活）

```go
func init() {
    worker.NewCronBuilder("my:cron:type", "我的定时任务").
        WithParamsHandler(handleMyCronParams).
        Register()
}
```

## 3. Task发送

### TaskBuilder（链式调用）

```go
package task

import (
    "context"
    "devinggo/modules/system/pkg/worker"
    "time"
)

type MyTaskData struct {
    Message string `json:"message"`
}

// 简单发送
func SendMyTask(ctx context.Context, message string) error {
    return worker.NewTaskBuilder(ctx, "my:task:type").
        WithData(MyTaskData{Message: message}).
        Send()
}

// 带延迟和队列发送
func SendMyTaskWithDelay(ctx context.Context, message string, delay time.Duration) error {
    return worker.NewTaskBuilder(ctx, "my:task:type").
        WithData(MyTaskData{Message: message}).
        WithDelay(delay).
        WithQueue("critical").
        WithTaskID("unique-id-123").
        Send()
}
```

## 5. 优势总结

### 核心优势

1. **代码量减少90%+**：从30-40行减少到1-2行
2. **无需定义结构体**：直接使用函数注册
3. **无需实现接口方法**：自动实现所有必需方法
4. **更易维护**：逻辑集中，不分散在多个方法中
5. **更易理解**：一眼看懂注册的是什么
6. **支持两种风格**：函数式和Builder模式都支持

### 无需样板代码

不需要：
- ❌ 定义结构体
- ❌ GetType()方法
- ❌ GetPayload()方法
- ❌ GetDescription()方法
- ❌ GetCronTask()方法
- ❌ 构造函数

## 7. 最佳实践

1. **优先使用函数式注册**：代码最简洁
2. **需要灵活性时使用Builder**：可以动态构建
3. **保持函数命名规范**：execute开头表示Worker，handle开头表示参数处理
4. **合理组织文件结构**：Worker放server目录，Cron放cron目录，Task放task目录
5. **使用类型安全的数据结构**：定义清晰的Data结构体
6. **适当的日志记录**：使用glob2.WithWorkLog()记录关键信息

## 5. 优势总结

### 新方式的优势

1. **代码量减少90%+**：从30-40行减少到1-2行
2. **无需定义结构体**：直接使用函数注册
3. **无需实现接口方法**：自动实现所有必需方法
4. **更易维护**：逻辑集中，不分散在多个方法中
5. **更易理解**：一眼看懂注册的是什么
6. **支持两种风格**：函数式和Builder模式都支持

### 旧方式的问题

1. **样板代码多**：每个Worker/Cron都需要定义结构体
2. **方法冗余**：GetType()、GetPayload()等方法重复
3. **维护成本高**：修改逻辑需要在多处改动
4. **代码分散**：逻辑分散在多个方法中

## 6. 迁移指南

### 如何迁移现有代码

旧代码：
```go
type cMyWorker struct {
    Type string
}

func NewMyWorker() *cMyWorker {
    return &cMyWorker{Type: "my:type"}
}

func (s *cMyWorker) GetType() string {
    return s.Type
}

func (s *cMyWorker) Execute(ctx context.Context, t *asynq.Task) error {
    // 执行逻辑
    return nil
}

func init() {
    worker.Register(NewMyWorker())
}
```

新代码：
```go
func init() {
    worker.RegisterWorkerFunc("my:type", executeMyWorker)
}

func executeMyWorker(ctx context.Context, t *asynq.Task) error {
    // 执行逻辑（原Execute方法的内容）
    return nil
}
```

### 迁移步骤

1. 将Execute方法的内容提取为独立函数
2. 使用RegisterWorkerFunc或RegisterCronFunc注册
3. 删除结构体定义和构造函数
4. 测试验证功能正常

## 9. 常见问题

**Q: 旧代码还能继续使用吗？**  
A: 可以，新旧方式完全兼容，可以逐步迁移。

**Q: 性能有影响吗？**  
A: 没有，新方式只是封装了注册过程，运行时性能完全一致。

**Q: 如何在参数处理函数中访问其他服务？**  
A: 参数处理函数可以调用任何服务或工具函数，没有限制。

**Q: 可以动态注册吗？**  
A: 可以，不一定要在init中注册，任何时候都可以调用注册函数。
